# KRAIL 1.1.0 — Knowledge Operations Foundations

KRAIL 1.1 makes knowledge operations explicit and inspectable while preserving
the 1.x local-runtime contract.

## Included

- typed action contracts with dry-run-first execution
- read-only retriever contracts and deterministic query plans
- retrieval v2 with reciprocal-rank fusion and stable trace digests
- evidence packets with citations, trust, freshness, gaps, and next actions
- `trigger` as preferred vocabulary with full `listener` compatibility
- unified workflow and agent run inspection
- bundled agent-readable guidance through `krail docs`
- version diagnostics in `doctor` and versioned `AGENTS.md` scaffolding
- additive SDK, CLI, and experimental MCP surfaces

## Compatibility

Existing 1.x projects do not require a schema migration. Existing `search`,
`think`, `listener`, task, and workflow commands remain supported. Search keeps
its existing result fields and adds query-plan, trace, and evidence-packet
fields. The MCP stable-v1 contract is unchanged; new 1.1 tools begin as
experimental.

## Intentionally Deferred

- crash-safe append-only workflow event sourcing
- arbitrary third-party action loading and remote plugin registries
- hosted multi-user authorization and operating-system sandboxing
- model-trained rerankers and calibrated relevance probabilities
- source-specific ingestion profiles beyond the existing source pipeline
