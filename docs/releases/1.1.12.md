# KRAIL 1.1.12 — Scalable Dataset Catalogs

KRAIL 1.1.12 adds a local-first path for working with large structured data
without treating every source row as an ontology individual.

## Highlights

- `sources/datasets.yaml` declares authoritative structured sources, stable
  keys, refresh policies, optional indexes, and semantic projections.
- `krail --local datasets cache-build` builds indexed DuckDB tables under
  `artifacts/data/`; cache state preserves source and schema fingerprints.
- `datasets cache-validate` is a release-safe freshness and structural gate;
  `datasets cache-benchmark` records latency for a project’s real dataset.
- `query routed` returns bounded, read-only results with backend provenance;
  `query source` reads a declared SQLite source directly.
- Hydration steps can use batches, watermarks, and a persistent entity
  resolution cache when a semantic projection must be updated incrementally.

The existing hydration and local-runtime contracts remain backward compatible.
