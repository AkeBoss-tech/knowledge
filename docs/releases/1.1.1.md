# KRAIL 1.1.1 — Portable Agent Workflows

KRAIL 1.1.1 keeps KRAIL workflows independent from a single agent vendor.

## Added

- `krail workflow import-claude` translates the safe, documented Claude dynamic
  workflow subset into repository-backed `krail-flow/v1` specifications.
- `krail workflow suggest-runner` reports the selected local harness for every
  workflow agent step, including availability and fallback rationale.
- Projects can register custom prompt-taking CLIs under `agents.harnesses` in
  `rail.yaml`; they participate in runner policy, dry-run dispatch, and
  workflow validation.

## Safety Boundary

The importer does not execute JavaScript. It accepts `meta`, `agent`, and
`pipeline` forms, while rejecting imports, process or filesystem access,
dynamic evaluation, and unsupported expressions.
