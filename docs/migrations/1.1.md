# Migrating to KRAIL 1.1

KRAIL 1.1 is additive. Upgrade the runtime and MCP adapter together, then run:

```bash
krail --version
krail --local doctor
```

`doctor` reports the imported runtime version and the `krail` executable found
on `PATH`, warning when they disagree.

## Vocabulary Changes

Use `trigger` in new documentation and automation. Existing `listener`
commands, files, and APIs continue to work and address the same records.

Use `action` for typed operations and `retriever` for read-only evidence
producers. Existing direct commands remain supported; actions add a reusable
contract rather than replacing those commands in 1.1.

## Search Consumers

Existing result fields remain available. Consumers may optionally adopt:

- `query_plan`
- `retrieval_trace`
- `evidence_packet`
- explicit `trust_state`, `source_type`, `freshness`, `citation`, and `context`

Do not compare retrieval-v2 scores as absolute probabilities. They are stable
ranking signals within a result set.

## MCP Consumers

The stable v1 tool groups are unchanged. New action, retriever, evidence,
trigger, and unified-run tools are experimental in 1.1 and may evolve within
the 1.x line before promotion into the stable contract.
