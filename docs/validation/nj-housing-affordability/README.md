# NJ Housing Affordability and Labor Market Study

Validation archetype: **time-series econ** (FRED hydrate → DuckDB → grounded research → expansion tasks).

## Live E2E runbook (Sprint 2)

### Prerequisites

| Requirement | Notes |
|-------------|--------|
| `GOOGLE_API_KEY` | Gemini planner + research (`run_live_agent_loop.py`) |
| `FRED_API_KEY` | Real FRED HTTP hydrate in phase 5 |
| `CONVEX_URL` / deploy key | Only for API + autopilot worker path (not required for the script) |
| RAIL API | `http://localhost:8000` — optional for full autopilot; script runs file-local |

### Quick path (no API)

From `packages/api/`:

```bash
python scripts/run_live_agent_loop.py
```

Phases 1–9: bootstrap repo, FRED hydrate, Gemini research, post-run audits, follow-up expansion tasks.

**Expected closeout:** `closeout` auditor may show `blocked` with `N non-terminal task(s) remain` until expansion tasks (`requires_expansion` / `blocked_by_data`) are completed or cancelled — this is correct fail-closed behavior.

Results summary: `docs/validation/live_agent_loop_summary.json`

### Full path (API + Convex + autopilot)

1. Start API: `uvicorn app.main:app --reload` (from `packages/api`)
2. Register project in Convex with `localRepoPath` → this directory
3. Set `FRED_API_KEY` in project secrets vault
4. `POST /jobs` hydrate or `rail hydrate` for default FRED pipeline
5. Run autopilot tick / agent sessions until all five auditors `ready`

### Verification

```bash
./scripts/run-verification.sh
python scripts/verify_project_state.py
```
