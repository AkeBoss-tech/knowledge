# Spec Update Plan (Work Orders)

To bridge the gap between our current architecture and the agentic vision, we need to systematically update our specifications (`specs/`) to include the Agent Layer.

Here are the work orders for updating the specs:

## Work Order 1: Introduce the Agent Orchestration Layer
**Target Spec:** `specs/architecture.md`
**Objective:** Add a new top-level component to the architecture: the Agent Orchestrator.
**Changes Needed:**
- Define where the LLM integration lives (e.g., a new `packages/agents/` directory or an extension to `packages/api/`).
- Diagram the communication flow: User Prompt -> Agent Orchestrator -> Code Generation -> Pipeline Execution -> Convex/Next.js.

## Work Order 2: Specify the AI Data Engineer Pipeline
**Target Spec:** `specs/engine.md` & `specs/yaml-config.md`
**Objective:** Document how the engine accepts AI-generated configs and handles dynamic schema extensions.
**Changes Needed:**
- Define the API contract for the AI Data Engineer to submit new data sources.
- Specify how the engine handles "Proposed Schema Extensions" without breaking the robust core ontology.
- Document a feedback loop: if the engine fails to parse data using the AI's config, how the error is fed back to the agent for self-reflection and fixing.

## Work Order 3: Specify the AI Analyst Sandbox
**Target Spec:** `specs/plugins.md` (Rename/expand to `specs/analyst-agents.md`)
**Objective:** Detail the execution environment for AI-generated analysis code.
**Changes Needed:**
- Define a secure execution sandbox for AI-generated Python (`statsmodels`, `linearmodels`, `pandas`).
- Document the API the "Analyst Agent" uses to query the populated ontology.
- Detail the structured output format the agent must adhere to so the results can be saved to the database.

## Work Order 4: Dynamic UI Deployment Spec
**Target Spec:** `specs/frontend.md`
**Objective:** Document how AI Analyst Agents deploy custom UIs.
**Changes Needed:**
- Define a "Dynamic Component Registry" in Next.js that can render React code generated by the agent.
- Alternatively, define a structured layout JSON schema that the agent emits, which the frontend engine interprets to render dashboards.
- Document how the agent creates and provisions new Convex tables if a custom dashboard requires persistent state.
